## ⚡ 每日開發流程 (SOP)

為了避免「悲劇」（檔案衝突），請大家養成習慣，每次開工前都要唸一遍口訣：**「先拉，再做，後推」**。

### 1️⃣ 開工第一件事：拉取 (Pull)
**只要你剛坐下準備開始畫圖，不管你覺得有沒有人改過，請先執行：**
```bash
git pull origin main

```

* **為什麼？** 這是把雲端最新的進度同步到你的電腦。如果你不拉就開始改，最後上傳時會發生衝突。

### 2️⃣ 進行開發

* 打開 KiCad，編輯你的電路圖或 PCB。
* 記得隨時存檔 (Ctrl+S)。

### 3️⃣ 收工上傳三步曲：加、存、推

當你完成一個階段的工作（例如：畫完電源線、放完電容），請依序執行：

**Step A: 加入變更 (Add)**
告訴 Git 這一輪你要上傳哪些檔案。

```bash
git add .

```

**Step B: 提交紀錄 (Commit)**
寫下你到底改了什麼。
！！！絕對不要有啥都沒有描述的commit！！！

```bash
git commit -m "清楚的描述，例如：完成 RocketCom 的 3.3V 電源佈線"

```

**Step C: 推送到雲端 (Push)**
這一步才是真的傳到 GitHub 上。

```bash
git push origin main

```

## 💣 KiCad 協作特別注意事項 (必讀！)

### 1. 關於 PCB 檔案 (`.kicad_pcb`)

Git 很擅長合併程式碼，但**非常不擅長合併 PCB 檔**。

* **黃金法則**：同一時間，**只能有一個人**修改同一張 PCB 板。
* **溝通**：要改 Layout 之前，請在群組喊一聲：「我要改通訊板的 PCB 囉！」，確認沒人正在改再動手。

## ❓ 常見問題救生圈

### Q1: 我輸入 `git push` 時出現 `error: failed to push some refs...` 怎麼辦？

這代表有人在你工作的時候，已經先上傳了新版本。
**解法：**

1. 先輸入 `git pull origin main` (把對方的東西抓下來合併)。
2. 如果不幸出現衝突 (Conflict)，請找組長求救。
3. 合併成功後，再輸入一次 `git push origin main`。

### Q2: 為什麼有些備份檔 (`.bak`) 傳不上去？

這是正常的！我們設定了 `.gitignore` 來過濾掉那些垃圾檔案，保持專案乾淨。請不要強行上傳它們。

* **KiCad 版本**：請大家統一使用 **KiCad 7.0** ，避免檔案格式跑掉。
* **Git 設定**：第一次使用請設定你的名字和 Email：
```bash

git config --global user.email "你的Email"
```
